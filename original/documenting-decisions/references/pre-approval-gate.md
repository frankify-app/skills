# Pre-approval gate

Before writing implementation code, produce a PLAN.md with:

- Files you intend to create or modify
- Any new dependencies, patterns, or abstractions you plan to introduce
- How you interpreted the task (especially any ambiguities you resolved)
- Any operations that would be irreversible

Wait for human approval of PLAN.md before proceeding. If the task is routine (all changes follow existing patterns, no new dependencies, no ambiguity), note "Routine task, no pre-approval needed" and proceed.
