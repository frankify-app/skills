# Scope interpretation

When a task is ambiguous, do not silently pick an interpretation. Instead:

1. State the ambiguity explicitly
2. List the possible interpretations (2-3 max)
3. State which one you would choose and why
4. Proceed with your chosen interpretation but flag it in the decision log as part of the pull-request description under SCOPE

If the task references concepts or components you cannot find in the codebase, stop and flag this rather than inventing something.
